# transaction-processor

A desktop app for importing and analyzing Chase bank transactions. Built with Tauri 2, SolidJS, and SQLite.

## Features

- Import Chase transaction CSV exports
- Duplicate detection on import
- Filter transactions by date, description, and amount
- Generate spending reports

## Requirements

### System

- **Rust** (stable toolchain) — install via [rustup](https://rustup.rs/)
- **Node.js** 18+ and **pnpm** — install pnpm via `npm install -g pnpm`
- **Tauri system dependencies** for your OS:
  - **Linux**: `libwebkit2gtk-4.1-dev`, `libgtk-3-dev`, `libayatana-appindicator3-dev`, `librsvg2-dev`, `patchelf`
    ```
    sudo apt install libwebkit2gtk-4.1-dev libgtk-3-dev libayatana-appindicator3-dev librsvg2-dev patchelf
    ```
  - **macOS**: Xcode Command Line Tools (`xcode-select --install`)
  - **Windows**: [Microsoft C++ Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) and [WebView2](https://developer.microsoft.com/en-us/microsoft-edge/webview2/)

## Setup

```sh
# Install frontend dependencies
pnpm install

# The Rust toolchain is pinned via rust-toolchain.toml — rustup will
# automatically download it on first use.
```

## Development

```sh
# Run the app in dev mode (hot-reload frontend + Rust backend)
pnpm tauri dev
```

The first run compiles the Rust backend, which may take a minute.

> **Note:** `src/bindings.ts` is auto-generated by tauri-specta when the app
> starts in debug mode. Do not edit it manually. If you change a backend command
> signature, run `cargo run` inside `src-tauri/` to regenerate it, then commit
> both the Rust changes and the updated `bindings.ts` together.

## Available Scripts

| Command | Description |
|---|---|
| `pnpm tauri dev` | Run app in development mode |
| `pnpm tauri build` | Build a release binary |
| `pnpm test` | Run frontend tests (Vitest) |
| `pnpm typecheck` | Type-check TypeScript |
| `pnpm lint` | Lint with ESLint |
| `pnpm format:check` | Check formatting with Prettier |

To run Rust tests:

```sh
cd src-tauri
cargo test
```
