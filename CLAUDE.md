# Claude Code Instructions

## System packages

If a task requires installing additional system packages (e.g. via `apt`, `brew`, or any system package manager), do not install them. Instead, stop and tell the user what package is needed so they can add it to the setup script in the Claude Code web interface.

## TypeScript bindings (src/bindings.ts)

Never edit `src/bindings.ts` manually. It is auto-generated by tauri-specta at runtime when `main()` runs in debug mode. To regenerate it, run `timeout 10 cargo run` inside `src-tauri/` — the specta export fires before the Tauri window opens, so the 10-second timeout is sufficient regardless of whether the environment is headless or has a display. In a headless environment GTK will panic after export (fine); in a GUI environment the window would otherwise stay open indefinitely. `cargo build` alone is not sufficient because it compiles but does not execute `main()`. When backend command signatures change, run `cargo run` in `src-tauri/`, then commit both the Rust changes and the updated `bindings.ts` together.

## Uncertainty

When unsure about a framework API, library behavior, or implementation detail — say so explicitly before writing code. Do not guess and do not silently fall back to patterns from similar frameworks (e.g., do not apply React patterns to Solid.js). Flag the uncertainty and either ask or look it up first.

## Notes

Use the `notes` directory when I ask you to write something to disk. Filenames should be `<YYYY-MM-DD>-<contextual filename>.md` where the date is today's date.
